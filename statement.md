#Problem statement
Small retail shops need a simple, reliable way to track inventory levels, record sales and purchases, and receive alerts when stock is running low. This system addresses that need with an easy-to-use console interface and persistent storage using CSV files.

#Scope of the project
Core inventory operations (add, update, list).

Recording transactions (sales and purchases).

Low stock alerting/report generation.

File-based persistence and reload.

Console UI for straightforward usage.

Excludes advanced features like barcode scanning, multi-user auth, and complex reporting.

#Target users
Small shop owners and staff who need to manage stock, prices, and transactions without complex software.

Students learning Core Java, OOP, collections, and file I/O.

#High-level features
Product catalog with ID-based lookup.

Transaction recording with validation (no overselling).

Low stock reporting.

CSV import/export of Products and Transactions.
